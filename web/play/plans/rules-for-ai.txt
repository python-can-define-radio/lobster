Here is our latest set of rules.

For all responses, follow these rules, which are preceded by definitions to clarify wording.

Definitions:
D1. Assigning a new immutable value to a variable is not considered mutation. For example, `var a = (b: 3, c: 5); a = (b: 7, c:9);` is not mutation; it is reassignment.
D2. Fields that are marked `final` cannot be reassigned. (This is intrinsic to Dart, but I am providing it as a reminder.)
D3. If we say something must be "notExtChg", that means it must be "not Externally Changeable". That includes the following:
 - immutable types (such as strings)
 - Stream (even though they are mutable, they are not mutable outside of where they are defined)
 - Observable (same explanation as Stream)

R10. If a function uses the `window` global JS object, it must be annotated with the `Eff` metadata. Note: when choosing the content of the `Eff` metadata, pick an arbitrary string that describes the side effect.
R11. The `document` global JS object should only occur in the following places: (a) the `HTML` class, (b) `main()`.
